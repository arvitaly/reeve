Reeve

Intervention tool for macOS, not a monitor. Observe, limit, release — then get back to work.

brew tap arvitaly/reeve && brew install --cask reeve
A reeve was a medieval estate manager — overseeing resources, keeping order.

Under threshold. Nothing to do.
Warning

What it does

Application grouping — all child processes under their parent app. One row per app, not per process.

Inline actions — hold-to-kill, suspend, lower priority, resume. Single click from the popover.

Rule engine — auto-kill or renice any app that exceeds a memory threshold. Sentence-syntax rule creation.

Diagnostics — per-app memory breakdown by VM region. Finds why Finder eats 3 GB (4000 Desktop files), why Chrome leaks (25 renderer processes), suggests remediations.

System pressure bar — kernel-reported RAM usage, not a sum of resident pages. Shows what the OS actually sees.

Menu bar severity — lights up when something crosses a threshold. Shows the offending app icon and memory value.

Widget modes — compact, expanded, pinned apps, dashboard with sparklines. Floating always-on-top overlay.

Disk analysis — per-app I/O rates, DerivedData size detection, cache cleanup suggestions.

No network I/O (not exposed by any public macOS API)
No GPU memory
No root-required actions

Diagnostics probes

VM Region Inspector — PROC_PIDREGIONPATHINFO breakdown for any app. Categories: MALLOC, Stack, Frameworks, CoreAnimation, CGImage, JavaScriptCore, SQLite.

Browser tabs — counts renderer child processes. Chrome, Brave, Edge, Arc, Vivaldi, Opera, Electron apps (Slack, Discord).

Finder Desktop — file count + QuickLook thumbnail cache size. Suggests moving files when Desktop exceeds 50 items.

Xcode DerivedData — measures ~/Library/Developer/Xcode/DerivedData. Actionable above 20 GB.

Docker memory — reads Docker settings.json memoryMiB. Flags allocation above 4 GB.

VS Code extensions — counts installed extensions. Advisory above 30.

Safari tabs — AppleScript tab counter with honest permission handling.

Self-probe — Reeve diagnoses its own memory. If it can't, the framework is wrong.

Design constraints

Single process
No daemon. No LaunchAgent. No root helper. Everything Reeve does, it does within its own process space.
Metric honesty
Every system call is traceable to a public source: man page, header, or Apple documentation. If a metric requires a private API, it is not exposed. "Unknown" is a valid value.
Mutation semantics
Reversible (renice, suspend) vs irreversible (SIGKILL, cache purge). The distinction is structural. Preflight names the class explicitly before any mutation.
Self-referential
Reeve must appear in its own top-consumer list. If it does not, the list is wrong or Reeve is hiding itself.

Architecture

ReeveKit

Library target. Process sampling via libproc, action domain, rule engine, diagnostic probes, VM region inspector. Fully unit-tested, no AppKit dependency.

Reeve

Executable target. SwiftUI + AppKit. MenuBarExtra, NSPanel overlay, application grouping via NSWorkspace, remediation execution.

macOS 13 Ventura or later. Apple Silicon. swift build from a clean clone produces a working binary.

Build from source

git clone https://github.com/arvitaly/reeve.git
cd reeve
make run

Requires Xcode command-line tools. No other dependencies.
